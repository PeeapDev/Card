# Register

Create a new user account and receive authentication tokens.

## Endpoint

```
POST /auth/register
```

## Request Body

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `email` | string | Yes | User's email address |
| `password` | string | Yes | Password (min 8 chars, must include uppercase, lowercase, number, special char) |
| `firstName` | string | Yes | User's first name |
| `lastName` | string | Yes | User's last name |

## Example Request

```bash
curl -X POST https://api.paymentsystem.com/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "user@example.com",
    "password": "SecurePass123!",
    "firstName": "John",
    "lastName": "Doe"
  }'
```

## Response

### Success (201 Created)

```json
{
  "user": {
    "id": "usr_abc123",
    "email": "user@example.com",
    "firstName": "John",
    "lastName": "Doe",
    "kycStatus": "NOT_STARTED",
    "kycTier": 0,
    "createdAt": "2024-01-15T10:30:00Z"
  },
  "tokens": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expiresIn": 900,
    "tokenType": "Bearer"
  }
}
```

### Errors

| Status | Code | Description |
|--------|------|-------------|
| 400 | `VALIDATION_ERROR` | Invalid request parameters |
| 409 | `EMAIL_EXISTS` | Email already registered |
| 422 | `WEAK_PASSWORD` | Password doesn't meet requirements |

```json
{
  "statusCode": 409,
  "message": "Email already exists",
  "error": "EMAIL_EXISTS"
}
```

## Password Requirements

- Minimum 8 characters
- At least one uppercase letter (A-Z)
- At least one lowercase letter (a-z)
- At least one number (0-9)
- At least one special character (@$!%*?&)

## Notes

- A default wallet in USD is automatically created for new users
- New users start at KYC Tier 0 with basic transaction limits
- Email verification may be required depending on configuration
