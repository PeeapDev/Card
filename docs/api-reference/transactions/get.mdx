# Get Transaction

Retrieve details for a specific transaction.

## Endpoint

```
GET /transactions/{transactionId}
```

## Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `transactionId` | string | Yes | The transaction ID |

## Headers

| Header | Required | Description |
|--------|----------|-------------|
| `Authorization` | Yes | Bearer access token |

## Example Request

```bash
curl https://api.paymentsystem.com/v1/transactions/txn_abc123 \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN"
```

### Errors

| Status | Code | Description |
|--------|------|-------------|
| 401 | `UNAUTHORIZED` | Invalid or missing access token |
| 403 | `FORBIDDEN` | Transaction belongs to another user |
| 404 | `NOT_FOUND` | Transaction does not exist |

```json
{
  "statusCode": 404,
  "message": "Transaction not found",
  "error": "NOT_FOUND"
}
```

## Response Fields

### Merchant Object

| Field | Description |
|-------|-------------|
| `id` | Merchant identifier |
| `name` | Merchant name |
| `category` | MCC (Merchant Category Code) |
| `categoryName` | Human-readable category name |
| `city` | Merchant city |
| `country` | Merchant country (ISO 3166-1 alpha-2) |

### Timeline Object

| Field | Description |
|-------|-------------|
| `status` | Transaction status at this point |
| `timestamp` | When this status was reached |

## Notes

- Transaction details include full merchant information
- Timeline shows the complete status history
- For cross-currency transactions, check `exchangeRate` and `billingAmount`
